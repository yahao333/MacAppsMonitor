# 自动开发规则 (Rules)

在为此项目编写或修改任何代码时，必须严格遵守以下规则：

## 1. 语言规范
- **代码注释**: 所有新增或修改的代码注释必须使用 **中文**。
- **日志信息**: 控制台调试信息（Debug Logs）必须使用 **中文** 或 **中英结合**，确保开发者能快速定位问题。

## 2. 注释保留原则
- 除非该段代码被彻底删除，否则严禁删除原有的代码注释。
- 修改逻辑时，应同步更新相关注释以保持一致性。

## 3. 调试要求
- 所有的异步请求（API Call）、本地存储读写、关键状态变更（State Change）必须添加 `console.debug` 或 `console.log` 输出详细的调试信息。
- 调试信息应包含：动作名称、参数、结果（成功/失败原因）。

## 4. 功能核对
- 任何修改在提交前，必须对照 `features.md` 检查是否意外破坏或删除了现有功能。
- 如果修改涉及到功能变更，必须同步更新 `features.md`。

## 5. API 规范
- 严格使用 `@google/genai` 规定的 SDK 调用方式。
- 使用 `process.env.API_KEY` 获取密钥，不得在代码中硬编码。
